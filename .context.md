# Cluely Pro — LLM Context

> Stealth meeting copilot. Invisible overlay that captures audio (speaker + mic), transcribes via Whisper, detects questions, and streams AI answers. Screen-capture-proof.

## Stack

| Layer | Tech | Entry |
|-------|------|-------|
| Desktop | Electron 30 (transparent, frameless, alwaysOnTop, contentProtection) | `main.js` |
| Renderer | Vanilla HTML/CSS/JS, marked.js for markdown | `renderer/index.html`, `renderer/style.css` |
| Preload | contextBridge IPC bridge | `preload.js` |
| Backend | Python 3, spawned as child process via `--ipc` flag | `backend/main.py` |
| Server | Next.js 14, Supabase, Razorpay | `server/` |

## Python Backend (`backend/`)

**Pipeline:** `AudioCapture → AudioChunker (VAD) → Transcriber (Whisper) → ContextManager → Assistant (LLaMA)`

| File | Purpose | Key Details |
|------|---------|-------------|
| `main.py` | Orchestrator (`CopilotPipeline`) | Two modes: `--ipc` (JSON-line stdin/stdout for Electron) or terminal (colored stderr). Dual audio: loopback + mic simultaneously, both feed one `chunk_queue`. |
| `audio_capture.py` | Audio input | `mode="loopback"` (WASAPI/pyaudiowpatch), `"mic"` (sounddevice), `"auto"` (try loopback then mic). 16kHz mono int16 PCM. 30ms frames (480 samples). Resamples from native rate. |
| `chunker.py` | VAD-based splitting | WebRTC VAD (aggressiveness 2-3) or energy fallback. Silence=500ms→flush, min=1s, max=15s. Outputs temp `.wav` files. 300ms pre-speech buffer. |
| `transcriber.py` | Speech-to-text | Groq API, `whisper-large-v3-turbo`. Primary + fallback API keys. ~300-500ms latency. |
| `context_manager.py` | Rolling transcript + question detection | 120s sliding window of `TranscriptSegment`s. 100+ trigger phrases + `?` detection. 8s cooldown. `add_transcript()` returns `{context, question, is_question}`. `force_question(lookback_seconds=10)` for manual trigger. |
| `assistant.py` | Answer generation | Groq `llama-3.3-70b-versatile`, temp=0.3, max 512 tokens. Streaming. System prompt optimized for small overlay (brief answers, MCQ handling, code snippets). Primary + fallback keys. |
| `screenshot.py` | Vision analysis | Groq `llama-4-scout-17b-16e-instruct`. Base64-encodes image → streams answer. |

**IPC Protocol** (JSON-line on stdout, one object per line):

```
Backend → Electron:
  {type:"transcript", text}
  {type:"question", text}
  {type:"answer_start", question}
  {type:"answer_chunk", text}
  {type:"answer_done", text, latency_ms}
  {type:"answer_error", text}
  {type:"listening", state:"ON"|"OFF"}
  {type:"status", text}

Electron → Backend (stdin):
  {command:"toggle"}
  {command:"force_answer"}
  {command:"screenshot", image_path}
  {command:"chat", prompt}
  {command:"set_user_context", user_context}
  {command:"quit"}
```

**Dependencies:** `sounddevice`, `pyaudiowpatch`, `soundfile`, `numpy`, `groq`, `pynput`, `python-dotenv`, `webrtcvad` (optional)

## Electron (`main.js`, `preload.js`)

**Window:** 420×340, transparent, frameless, alwaysOnTop (screen-saver level), skipTaskbar, non-focusable (toggled for settings/chat input), `setContentProtection(true)`.

**Hotkeys:** `Ctrl+Shift+Z` toggle visibility, `Ctrl+Shift+P` spawn/reopen, `Ctrl+G` screenshot, `Ctrl+Shift+A` force answer, `Ctrl+Shift+L` toggle listening.

**Backend spawn:** Packaged → `resources/backend/cluely-backend.exe`; Dev → `python3 backend/main.py --ipc`. Env vars: `GROQ_API_KEY`, `CLUELY_USER_CONTEXT`, `PYTHONIOENCODING=utf-8`.

**preload.js** exposes `window.copilot`: `onMessage()`, `sendCommand()`, `copyToClipboard()`, `setIgnoreMouse()`, `closeWindow()`, `setFocusable()`, `get/saveApiKey()`, `get/saveUserContext()`, `get/saveLicenseKey()`, `getLicenseStatus()`, `openExternal()`.

## Renderer (`renderer/`)

**Layout:** Drag bar (status + buttons) → Transcript section (30%, auto-scroll, 50 chunks max) → Divider → Answer section (flex:1, markdown rendered, copy buttons on code blocks, streaming cursor) → Chat input bar.

**Settings overlay:** License key, Groq API key, user context textarea (1000 chars). Persisted via Electron IPC.

**Styling:** Dark glassmorphic (`rgba(12,12,15,0.55)`, backdrop blur 24px), accent `#6ee7b7`, 12px base font, JetBrains Mono for code.

## Server (`server/`)

**Next.js 14 app** deployed to Vercel. Supabase Postgres for license storage.

**DB schema** (`licenses` table): `id`, `license_key` (CLP-XXXXX-XXXXX-XXXXX), `email`, `plan` (free/pro/promax), `hwid` (SHA-256 device binding), `usage_answers`, `usage_screenshots`, `created_at`, `expires_at`, `razorpay_*`, `active`.

**Plans:**
- free: 10 answers, 5 screenshots, no expiry
- pro: unlimited, 30 days, ₹499
- promax: unlimited, 365 days, ₹2999

**API routes:**
- `POST /api/license/create-free` — email → license_key
- `POST /api/license/validate` — license_key + hwid → JWT token + usage remaining
- `POST /api/license/usage` — increment answer/screenshot counter
- `POST /api/payment/create-order` — Razorpay order for pro/promax
- `POST /api/payment/verify` — HMAC-SHA256 verification → create license

**Auth:** HS256 JWT via `jose`, 6h expiry. HWID = `SHA256(hostname+username+cpu+os)`.

## Key Patterns

- All Groq API calls use primary + fallback key for rate-limit resilience
- Audio pipeline uses thread-safe queues between stages
- Two separate `AudioChunker` instances (one per audio source) keep VAD state independent
- Backend errors are non-fatal where possible (audio failure → screenshot-only mode)
- Window is non-focusable by default, temporarily made focusable for text input
